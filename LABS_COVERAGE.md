# Как приложение покрывает лабораторные работы

## Краткий обзор

Приложение **«Гид по Хабаровску»** реализовано как одно Android-приложение на Kotlin с классическими **Views/XML**. В нём есть список достопримечательностей, экран подробностей, карта OpenStreetMap, избранное, заметки, настройки, виджет, уведомления, аудио, видео, камера и облачная рекомендация из Firebase Realtime Database.

Такой формат позволяет закрыть все темы лабораторных работ в одном связном приложении, а не создавать отдельный проект для каждой работы.

## Лабораторная работа №1. Создание первого приложения

Покрывается самим Android-проектом `CityGuide`: проект создан, запускается на эмуляторе/устройстве и имеет собственное имя приложения.

ФИО и группа студента находятся в диалоговом окне **«О программе»**:

```text
Панченко Владимир Юрьевич
МО911ИВС
```

Технически это реализовано через строковый ресурс `about_text` в `res/values/strings.xml` и `AlertDialog` в `MainActivity`.

## Лабораторная работа №2. Работа с элементами

На главном экране используются базовые элементы интерфейса:

- `TextView` для вывода результата поиска;
- `EditText` для ввода запроса;
- `Button` для запуска поиска;
- `RecyclerView` для отображения списка мест.

После ввода текста и нажатия кнопки **«Найти»** приложение фильтрует список достопримечательностей и выводит количество найденных мест.

Технически логика находится в `MainActivity`: метод `filterPlaces()` читает текст из `EditText`, фильтрует данные из `PlacesRepository` и обновляет адаптер списка.

## Лабораторная работа №3. Работа с экранами

Приложение использует несколько `Activity`:

- `MainActivity` — главный экран со списком мест;
- `PlaceDetailsActivity` — подробности выбранного места;
- `MapActivity` — карта;
- `SettingsActivity` — настройки;
- `VideoActivity` — видео;
- `CloudActivity` — облачная рекомендация;
- `SavedPlacesActivity` — избранное.

Переходы выполняются через `Intent`. Например, при выборе места из списка в `MainActivity` открывается `PlaceDetailsActivity`, а `place_id` передаётся через `putExtra`.

## Лабораторная работа №4. Стили и темы

В приложении настроена собственная тема и стили:

- основные цвета города заданы в `res/values/colors.xml`;
- тема приложения задана в `res/values/themes.xml`;
- кнопки используют стили `CityButton` и `CityOutlineButton`;
- карточки используют фон `card_background.xml`.

Это делает интерфейс нестандартным и единообразным: зелёная панель, светлый фон, карточки мест и одинаковые кнопки.

## Лабораторная работа №5. Работа со списками

Списки реализованы через `RecyclerView`.

В приложении есть несколько списков:

- список достопримечательностей на главном экране;
- список избранных мест;
- список заметок на экране места.

Для списка мест используется `PlaceAdapter`, для избранного — `SavedPlaceAdapter`, для заметок — `NoteAdapter`.

Каждый элемент списка места содержит изображение, название, категорию и краткое описание.

## Лабораторная работа №6. Работа с анимацией

Анимация используется при добавлении или удалении места из избранного.

На экране подробностей при нажатии кнопки **«Добавить в избранное»** или **«Убрать из избранного»** запускается стандартная Android-анимация:

```kotlin
AnimationUtils.loadAnimation(this, android.R.anim.fade_in)
```

Это демонстрирует простую tween-анимацию интерфейсного элемента.

## Лабораторная работа №7. Работа с картами

Карта реализована через OpenStreetMap с библиотекой OSMDroid.

Экран `MapActivity` содержит `MapView`, на котором отображаются маркеры всех мест Хабаровска из `PlacesRepository`.

Если открыть карту из экрана конкретного места, карта центрируется на этом месте. Если открыть карту с главного экрана, показывается общий вид города.

Для загрузки карты требуется интернет на устройстве или эмуляторе.

## Лабораторная работа №8. Создание виджета

В приложении есть домашний виджет **«Место дня»**.

Виджет объявлен в `AndroidManifest.xml` через `CityPlaceWidgetProvider`, а его параметры находятся в `res/xml/city_widget_info.xml`.

Макет виджета находится в `res/layout/widget_city_place.xml`. Он показывает:

- заголовок `Место дня`;
- место `Набережная Амура`;
- изображение набережной.

При нажатии на виджет открывается главное окно приложения.

## Лабораторная работа №9. Работа с меню

В приложении используются два вида меню:

1. Опциональное меню в верхней панели главного экрана:
   - `Избранное`;
   - `Облако`;
   - `Настройки`;
   - `О программе`.

2. Контекстное меню по долгому нажатию на место в списке:
   - `Добавить в избранное`.

Опциональное меню создаётся через XML-файл `res/menu/main_menu.xml`, а контекстное меню — через `res/menu/place_context_menu.xml`.

## Лабораторная работа №10. Работа с диалоговыми окнами

Диалоговые окна используются в нескольких местах:

- **«О программе»** показывает ФИО, группу и описание приложения;
- при удалении заметки показывается подтверждение;
- если отсутствует аудио или видео файл, показывается предупреждение.

Диалоги реализованы через `AlertDialog.Builder`.

## Лабораторная работа №11. Работа с уведомлениями

На экране подробностей есть кнопка **«Показать уведомление»**.

При нажатии приложение создаёт уведомление с текстом:

```text
Не забудьте посетить: <название места>
```

Для Android 8+ создаётся notification channel. Для Android 13+ в манифест добавлено разрешение `POST_NOTIFICATIONS`, а приложение запрашивает разрешение у пользователя.

## Лабораторная работа №12. Работа с аудио

На экране подробностей есть кнопка **«Включить аудиогид»**.

Приложение ищет файл:

```text
app/src/main/res/raw/audio_guide.mp3
```

Если файл есть, он проигрывается через `MediaPlayer`. Если файла нет, показывается диалог с инструкцией.

## Лабораторная работа №13. Работа с видео

Видео открывается с экрана подробностей кнопкой **«Открыть видео»**.

Используется отдельная `VideoActivity` с элементом `VideoView`. Приложение проигрывает файл:

```text
app/src/main/res/raw/city_video.mp4
```

Для управления видео используется `MediaController`. Экран видео зафиксирован в портретной ориентации, а видео выровнено по центру.

## Лабораторная работа №14. Работа с камерой

На экране подробностей есть кнопка **«Сделать фото»**.

При нажатии приложение запускает стандартное приложение камеры через:

```kotlin
Intent(MediaStore.ACTION_IMAGE_CAPTURE)
```

После съёмки камера возвращает thumbnail-изображение, которое показывается в `ImageView` на экране подробностей.

Фото не сохраняется постоянно. Это сделано в стиле лабораторной работы: сделать снимок и показать его в интерфейсе.

## Лабораторная работа №15. Работа с настройками и БД

Лабораторная работа покрыта двумя частями.

### SharedPreferences

Экран `SettingsActivity` сохраняет настройки через `SharedPreferences`.

Сейчас есть две настройки:

- `Напоминать о местах`;
- `Показывать посещённые места`.

После сохранения настройки остаются доступными после перезапуска приложения.

### SQLite

SQLite реализован в `SavedPlaceDbHelper`.

В базе данных есть:

- таблица `saved_places` для избранных мест;
- таблица `place_notes` для заметок.

Заметки имеют CRUD-логику:

- создать заметку;
- выбрать заметку;
- изменить заметку;
- удалить заметку;
- просмотреть все заметки выбранного места.

Избранное вынесено в отдельный экран и показывает только места, добавленные в избранное.

## Лабораторная работа №16. Облачное мобильное приложение

Облачная часть реализована через **Firebase Realtime Database**.

Экран `CloudActivity` читает данные по пути:

```text
/recommendation/title
/recommendation/text
```

Пример данных:

```json
{
  "recommendation": {
    "title": "Набережная Амура",
    "text": "Сегодня стоит прогуляться по набережной Амура."
  }
}
```

Если Firebase не настроен или данные отсутствуют, приложение показывает понятное сообщение вместо падения.

- анимацию;
- карты;
- виджет;
- меню;
- диалоги;

- видео;
