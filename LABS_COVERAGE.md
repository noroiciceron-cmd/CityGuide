# Как приложение покрывает лабораторные работы

## Краткий обзор

Приложение **«Гид по Хабаровску»** реализовано как Android-приложение на Kotlin с классическими **Views/XML**.

В приложении есть:

- главный экран со списком достопримечательностей;
- поиск по названию и категории;
- экран подробностей места;
- карта OpenStreetMap;
- избранное;
- заметки с сохранением в SQLite;
- пользовательские фотографии мест;
- настройки через SharedPreferences;
- домашний виджет **«Место дня»**;
- уведомления;
- аудиогид;
- видеоэкран;
- экран облачной рекомендации из Firebase Realtime Database;
- собственная иконка приложения.

Такой формат позволяет закрыть темы лабораторных работ в одном связном приложении, а не создавать отдельный проект для каждой темы.

---

## Лабораторная работа №1. Создание первого приложения

Покрывается самим Android-проектом `CityGuide`.

В проекте настроено приложение с собственным названием:

```text
Гид по Хабаровску
```

Также в приложении есть окно **«О программе»**, где указаны данные студента и краткое описание проекта:

```text
Панченко Владимир Юрьевич
МО911ИВС

Приложение-гид по достопримечательностям Хабаровска.
```

Технически данные находятся в строковом ресурсе `about_text` в файле:

```text
res/values/strings.xml
```

Диалог открывается из главного меню через `AlertDialog`.

---

## Лабораторная работа №2. Работа с элементами интерфейса

На главном экране используются базовые UI-элементы Android:

- `TextView` для заголовков и статуса поиска;
- `EditText` для ввода поискового запроса;
- `Button` для запуска поиска;
- `RecyclerView` для списка мест;
- `ImageView` для изображений достопримечательностей.

Поиск работает по названию и категории места. После ввода запроса приложение фильтрует список достопримечательностей и показывает результат на экране.

Также в проекте исправлены цвета полей ввода: текст в поиске и заметках отображается читаемо, а подсказки вынесены в `strings.xml`.

---

## Лабораторная работа №3. Работа с экранами

Приложение использует несколько `Activity`:

- `MainActivity` — главный экран;
- `PlaceDetailsActivity` — подробная карточка места;
- `MapActivity` — карта;
- `SettingsActivity` — настройки;
- `SavedPlacesActivity` — избранные места;
- `VideoActivity` — экран видео;
- `CloudActivity` — облачная рекомендация из Firebase.

Переходы между экранами выполняются через `Intent`.

Например, при выборе места из списка открывается `PlaceDetailsActivity`, а идентификатор места передаётся через `putExtra`.

---

## Лабораторная работа №4. Стили и темы

В приложении используется единый визуальный стиль:

- цвета заданы в `res/values/colors.xml`;
- тема приложения задана в `res/values/themes.xml`;
- кнопки используют стили `CityButton` и `CityOutlineButton`;
- карточки используют отдельный drawable-фон;
- новая иконка приложения заменяет стандартный Android-значок.

Интерфейс оформлен в единой цветовой схеме и выглядит как цельное приложение-гид.

---

## Лабораторная работа №5. Работа со списками

Списки реализованы через `RecyclerView`.

В приложении используются несколько списков:

- список достопримечательностей на главном экране;
- список избранных мест;
- список заметок в карточке места.

Для них используются адаптеры:

```text
PlaceAdapter
SavedPlaceAdapter
NoteAdapter
```

Каждая карточка места содержит изображение, название, категорию и описание.

---

## Лабораторная работа №6. Работа с анимацией

Анимация используется при добавлении или удалении места из избранного.

На экране подробностей при нажатии кнопки **«Добавить в избранное»** или **«Убрать из избранного»** запускается стандартная Android-анимация:

```kotlin
AnimationUtils.loadAnimation(this, android.R.anim.fade_in)
```

Это демонстрирует простую анимацию интерфейсного элемента.

---

## Лабораторная работа №7. Работа с картами

Карта реализована через OpenStreetMap с библиотекой OSMDroid.

Экран `MapActivity` содержит `MapView` и показывает маркеры достопримечательностей из `PlacesRepository`.

Если карта открывается из карточки конкретного места, приложение передаёт `place_id`, и карта центрируется на выбранной достопримечательности.

Если карта открывается с главного экрана, показывается общий вид города.

---

## Лабораторная работа №8. Создание виджета

В приложении есть домашний виджет **«Место дня»**.

Виджет реализован через:

```text
CityPlaceWidgetProvider
res/layout/widget_city_place.xml
res/xml/city_widget_info.xml
```

Новая логика виджета:

- выбирает случайное место из `PlacesRepository`;
- сохраняет выбранное место и дату в `SharedPreferences`;
- в течение одного дня показывает одно и то же место;
- на следующий день может выбрать новое место;
- отображает название и изображение места;
- по нажатию открывает карточку выбранной достопримечательности.

Это делает виджет не статичным, а связанным с реальными данными приложения.

---

## Лабораторная работа №9. Работа с меню

В приложении используются меню:

1. Опциональное меню в верхней панели главного экрана:

   - **Избранное**;
   - **Облако**;
   - **Настройки**;
   - **О программе**.

2. Контекстное меню для места в списке:

   - **Добавить в избранное**.

Меню позволяет перейти к дополнительным функциям приложения и управлять выбранными местами.

Диалоги используются в нескольких местах:

- окно **«О программе»**;
- подтверждение удаления заметки;
- сообщение об отсутствии аудиофайла;
- сообщение об отсутствии видеофайла.

```kotlin
AlertDialog.Builder
```

## Лабораторная работа №11. Работа с уведомлениями

На экране подробностей есть кнопка **«Показать уведомление»**.

При нажатии приложение создаёт уведомление с текстом:

```text
Не забудьте посетить: <название места>
```

Для Android 8+ создаётся `NotificationChannel`.

Для Android 13+ приложение проверяет и запрашивает разрешение:

```text
POST_NOTIFICATIONS
```

---

## Лабораторная работа №12. Работа с аудио

На экране подробностей есть кнопка **«Включить аудиогид»**.

Приложение ищет файл:

```text
app/src/main/res/raw/audio_guide.mp3
```

Если файл найден, он проигрывается через `MediaPlayer`.

Если файла нет, приложение показывает понятный диалог с инструкцией.

---

## Лабораторная работа №13. Работа с видео

Видео открывается с экрана подробностей через кнопку **«Открыть видео»**.

Используется отдельная `VideoActivity` с `VideoView` и `MediaController`.

Приложение проигрывает файл:

```text
app/src/main/res/raw/city_video.mp4
```

Также добавлена рабочая настройка **«Автоматически запускать видео»**:

- если настройка включена, видео запускается автоматически;
- если настройка выключена, видео открывается на паузе и ждёт действия пользователя.

Настройка сохраняется через `SharedPreferences`.

---

## Лабораторная работа №14. Работа с камерой

На экране подробностей есть кнопка **«Сделать фото»**.

Камера запускается через:

```kotlin
Intent(MediaStore.ACTION_IMAGE_CAPTURE)
```

Новая логика камеры улучшена по сравнению с базовым вариантом:

- приложение сохраняет полноценное фото, а не маленький thumbnail;
- фото создаётся через `FileProvider`;
- URI фото сохраняется в базе данных;
- для каждого места хранится отдельный список фотографий;
- пользователь может переключаться между фото кнопками **«Назад»** и **«Вперёд»**;
- отображается счётчик вида `Фото 1 из 3`;
- фото показывается без обрезания;
- по нажатию фото открывается во внешнем просмотрщике изображения.

Это покрывает работу с камерой, файловыми URI и отображением результата в интерфейсе.

---

## Лабораторная работа №15. Работа с настройками и SQLite

Лабораторная работа покрыта двумя частями.

### SharedPreferences

Экран `SettingsActivity` сохраняет настройку:

```text
Автоматически запускать видео
```

Ключ настройки:

```text
video_autoplay
```

Старые фиктивные настройки были удалены, а новая настройка реально влияет на поведение `VideoActivity`.

### SQLite

SQLite реализован в `SavedPlaceDbHelper`.

В базе данных используются таблицы:

- `saved_places` — избранные места;
- `place_notes` — заметки;
- `place_photos` — фотографии, связанные с конкретным местом.

Заметки поддерживают CRUD-логику:

- создать заметку;
- выбрать заметку из списка;
- изменить заметку;
- удалить заметку;
- просмотреть все заметки выбранного места.

Фотографии также сохраняются по `place_id`, поэтому фото одного места не смешиваются с фото другого места.

## Лабораторная работа №16. Облачное мобильное приложение

Облачная часть реализована через **Firebase Realtime Database**.

Экран `CloudActivity` читает данные по пути:

```text
/recommendation/title
/recommendation/text
```

Пример структуры базы:

```json
{
  "recommendation": {
    "title": "Набережная Амура",
    "text": "Сегодня стоит прогуляться по набережной Амура."
  }
}
```

Если Firebase не настроен, отсутствует `google-services.json`, неверный URL базы или данных нет по нужному пути, приложение показывает понятное сообщение вместо падения.

Рекомендуемые правила доступа для учебной версии:

```json
{
  "rules": {
    ".read": false,
    ".write": false,

    "recommendation": {
      ".read": true,
      ".write": false
    }
  }
}
```

Так приложение может читать облачную рекомендацию, а изменение данных выполняется вручную через Firebase Console.
